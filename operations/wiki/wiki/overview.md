# Operations Wiki — Overview

## Purpose
Second brain for the operations team. SOPs, runbooks, process docs — queryable so no one invents the wheel or panics during an incident.

## Three Moments
- **Process lookup:** Task description → correct SOP with owner and steps
- **Incident response:** Incident description → severity level + escalation path + checklist
- **Process creation:** Description → new SOP in standard format with owner and verification

## Current Operational State

- **Incident response:** Runbook v1.3 — P0-P3 severity levels, IC/TL/Comms roles, post-mortem format
- **On-call:** PagerDuty rotation active
