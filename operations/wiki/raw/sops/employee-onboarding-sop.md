# Employee Onboarding SOP

> Version 2.0 | Last updated: 2026-05 | Owner: Operations Manager
> Trigger: New employee starts on their first day

---

## Purpose

Ensure every new employee is set up, connected, informed, and productive within their first 30 days. A consistent onboarding experience reduces ramp time, reduces anxiety, and signals to the employee that the company is organized.

---

## Scope

This SOP applies to all full-time employees, both remote and in-office. Contractors have a separate lightweight version (see `raw/sops/contractor-onboarding-sop.md`).

---

## Owners & Roles

| Role | Responsibility |
|------|---------------|
| Operations Manager | Pre-boarding setup, access provisioning, SOP owner |
| HR Lead | Employment docs, compliance, benefits enrollment |
| Hiring Manager | Role-specific onboarding, 30-day check-in |
| IT / DevOps | Device setup, system access, security training |
| Buddy (assigned by manager) | Day-to-day support, culture guide, first 2 weeks |

---

## Pre-Boarding Checklist (1 week before start date)

### Operations Manager
- [ ] Send welcome email with:
  - Start date, time, location (or video link)
  - Who to ask for / who will meet them
  - Dress code if relevant
  - Parking / transport info if in-office
  - What to bring (ID for verification)
- [ ] Create accounts: email, Slack, Notion, GitHub (if engineering), Figma (if design), any domain-specific tools
- [ ] Set up device and ship (remote) or prepare desk setup (in-office)
- [ ] Add to relevant Slack channels: #general, #team-[domain], #onboarding
- [ ] Add to team calendar
- [ ] Assign onboarding buddy (confirm with buddy)
- [ ] Create 30-60-90 day plan doc (template in Notion → Onboarding)

### HR Lead
- [ ] Send employment contract for signature
- [ ] Collect signed offer letter, ID documents
- [ ] Set up payroll
- [ ] Enroll in benefits (send enrollment deadline and instructions)
- [ ] Add to HRIS system

### Hiring Manager
- [ ] Schedule Day 1 welcome call (30 min, just manager + new hire)
- [ ] Schedule Week 1 intro calls with key team members (15 min each)
- [ ] Draft first project or starter task for Week 2
- [ ] Brief the team: who is joining, what they'll work on, how to welcome them

---

## Day 1 Checklist

### Morning (first 2 hours)
- [ ] Welcome 1:1 with hiring manager (30 min)
  - Cover: role context, immediate priorities, how to ask for help
  - NOT a deep dive into strategy — keep it human and orienting
- [ ] Introduce to onboarding buddy
- [ ] Device and account access confirmed working
- [ ] Tour of physical/virtual office (where things live in Notion, Slack norms, key channels)

### Afternoon (remaining day)
- [ ] Complete mandatory compliance training (security, data privacy) — link in onboarding Notion page
- [ ] Complete tool setup checklist (password manager, 2FA on all accounts)
- [ ] Read company handbook (link in onboarding doc)
- [ ] Schedule Week 1 intro calls (if not pre-scheduled)
- [ ] First conversation with buddy — informal, just get to know each other

### End of Day 1 — Operations Manager confirms
- [ ] New hire has working access to all required systems
- [ ] New hire is in all correct Slack channels
- [ ] Compliance training started

---

## Week 1 Checklist

### New Hire
- [ ] Complete all intro calls with team members
- [ ] Read all role-relevant documentation (product docs, design system, process docs — provided by manager)
- [ ] Set up personal workspace (Notion/task system, communication preferences)
- [ ] Benefits enrollment completed (deadline is typically end of Week 1)
- [ ] Complete full compliance training

### Hiring Manager
- [ ] Day 3 check-in (15 min): how is it going? Any blockers?
- [ ] Confirm new hire has everything they need to start Week 2 work
- [ ] Introduce to any cross-functional partners

### Buddy
- [ ] Daily informal check-in (Slack message, 5 min)
- [ ] Walk through unwritten norms: how decisions get made, meeting culture, async vs. sync expectations

---

## Weeks 2-4 Checklist

### New Hire
- [ ] First real task or project delivered (defined by manager in 30-60-90 plan)
- [ ] Attend first team meeting and contribute (not just observe)
- [ ] Identify 2-3 things they're unclear about — document and resolve with manager

### Hiring Manager
- [ ] End of Week 2: async update on how onboarding is going
- [ ] End of Week 4: 30-day check-in (formal 1:1, 45 min)
  - What's going well?
  - What's confusing or unclear?
  - What resources or support would help?
  - Confirm 60-day goals

### Operations Manager
- [ ] Confirm all access and tools still working
- [ ] Remove new hire from "new hire" Slack label after 30 days

## 30-Day Check-In Template (for Hiring Manager)

```
New hire: [name]
Date: [date]
Manager: [name]

1. What has gone well in your first 30 days?

2. What has been harder than expected?

3. What's one thing you still don't fully understand yet?

4. What would help you be more effective in the next 30 days?

5. How are you finding the team / culture so far?

Notes for Operations:
- [ ] Any process gaps surfaced during onboarding → document in onboarding SOP update log
- [ ] Any tooling issues → ticket to IT
```

## Escalation

| Issue | Escalate to |
|-------|------------|
| Access not provisioned after Day 1 | Operations Manager immediately |
| New hire not receiving device (remote) | Operations Manager + IT |
| Compliance training not completed by end of Week 1 | HR Lead |
| New hire reports poor experience | Operations Manager + HR Lead |
| Conflict with buddy | HR Lead |

---

## Common Failure Points (from past onboardings)

1. **Account provisioning delayed** — operations manager must initiate 5+ business days before start date, not the day before.
2. **No buddy assigned** — buddy is critical for culture onboarding. Do not skip this.
3. **Manager too busy in Week 1** — minimum 3 check-ins in Week 1 is non-negotiable. Block this time before the start date.
4. **No defined first task** — "just observe and learn" for 2+ weeks is demotivating. New hire needs something to contribute to.
5. **Compliance training deferred** — it must be completed in Week 1. Link in welcome email.
